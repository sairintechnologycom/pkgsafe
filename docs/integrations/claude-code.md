# PkgSafe Claude Code Integration Guide

PkgSafe acts as the package safety guardrail layer for Claude Code, preventing the autonomous installation of risky, typosquatted, or malicious dependencies during coding loops.

## Purpose

Claude Code can connect to external tools through MCP servers. By configuring PkgSafe as an MCP server, Claude Code is forced to evaluate packages before installing or recommending them.

## MCP Configuration

To register the PkgSafe MCP server in Claude Code, add the following entry to your Claude Code MCP configuration file (`~/.claude/config.json` or equivalent configuration path):

```json
{
  "mcpServers": {
    "pkgsafe": {
      "command": "pkgsafe",
      "args": ["mcp", "serve"]
    }
  }
}
```

## Allowlisted PkgSafe Tools

Expose only the following safe read/check tools:

* `check_package`: Checks a package's safety against policy rules.
* `check_install_command`: Parses and guards shell install commands generated by Claude Code.
* `review_dependency_diff`: Inspects dependency changes in modified manifest files.
* `explain_policy_decision`: Provides remediation recommendations for failed rules.

> [!CAUTION]
> Do not expose broad shell execution or install execution through the MCP server. Only expose the narrow PkgSafe check, review, and explain tools.

## Example Package Check

```json
{
  "name": "check_package",
  "arguments": {
    "ecosystem": "npm",
    "name": "express",
    "version": "latest"
  }
}
```

### Expected Output

```json
{
  "decision": "ALLOW",
  "risk_score": 0,
  "confidence": "high",
  "top_reasons": [
    "No critical vulnerabilities found",
    "No lifecycle scripts detected",
    "Package age and project health acceptable"
  ],
  "agent_instruction": "Package may be installed.",
  "allowed_next_actions": ["proceed"],
  "prohibited_actions": []
}
```

## Claude-Specific Guardrail

* **Hard BLOCK**: Claude Code must never override `BLOCK` decisions.
* **No Lifecycle Executions for BLOCK**: Under no circumstances should Claude Code execute lifecycle scripts or setup commands for blocked packages.
* **Human Approval on WARN**: If a package is checked and returns `WARN`, Claude Code must ask the user for confirmation before proceeding with the install.

## Troubleshooting

1. **Tool discovery fails**: Ensure `pkgsafe` is installed and globally executable in your terminal environment.
2. **Offline mode**: If running in an air-gapped environment or to speed up checks, configure the command with the `--offline` flag:
   ```json
   "args": ["mcp", "serve", "--offline"]
   ```
