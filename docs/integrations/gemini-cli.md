# PkgSafe Gemini CLI Integration Guide

PkgSafe acts as the package safety guardrail layer for the Gemini CLI agent, preventing the autonomous installation of risky, typosquatted, or malicious dependencies during coding loops.

## Purpose

Gemini CLI supports configuring MCP servers to discover and execute external tools with structured responses. By configuring PkgSafe, Gemini CLI discovers security tools and enforces package safety guardrails.

## MCP Configuration

Add the PkgSafe MCP configuration to your Gemini CLI settings file (e.g. `settings.json` under the `mcpServers` block):

```json
{
  "mcpServers": {
    "pkgsafe": {
      "command": "pkgsafe",
      "args": ["mcp", "serve"]
    }
  }
}
```

## Allowlisted PkgSafe Tools

Expose only the following safe read/check tools:

* `check_package`: Checks a package's safety against policy rules.
* `check_install_command`: Parses and guards shell install commands generated by Gemini CLI.
* `review_dependency_diff`: Inspects dependency changes in modified manifest files.
* `explain_policy_decision`: Provides remediation recommendations for failed rules.

> [!CAUTION]
> Ensure only read and check tools are allowlisted. Never expose broad shell execution or package installation capabilities via MCP.

## Example Package Check

```json
{
  "name": "check_package",
  "arguments": {
    "ecosystem": "npm",
    "name": "left-pad",
    "version": "latest"
  }
}
```

### Expected Output

```json
{
  "decision": "WARN",
  "risk_score": 38,
  "confidence": "high",
  "top_reasons": [
    "Package has moderate risk score"
  ],
  "agent_instruction": "Do not install automatically. Ask the user for approval or choose an existing dependency.",
  "allowed_next_actions": [
    "ask_user",
    "suggest_alternative",
    "remove_dependency"
  ],
  "prohibited_actions": [
    "run_install",
    "execute_lifecycle_script"
  ]
}
```

## Gemini-Specific Guardrail

* **Hard BLOCK**: Gemini CLI must never execute commands or suggest packages that return `BLOCK`.
* **Autonomous Execution Restrictions**: If a command or package is flagged with `WARN` or `REVIEW_REQUIRED`, Gemini CLI must delegate decision-making to the user before running any install.

## Troubleshooting

1. **Server connection error**: Verify that the command path points to the correct `pkgsafe` binary and that standard streams (stdio) are not cluttered by non-JSON messages.
2. **Offline support**: Run with the offline flag if registry lookups are slow:
   ```json
   "args": ["mcp", "serve", "--offline"]
   ```
